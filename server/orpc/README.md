# oRPC Architecture

This project uses oRPC in OpenAPI mode only.

Public browser code uses `OpenAPILink` against `/api`, and the server exposes
the same router through `OpenAPIHandler`. Do not add `RPCLink`, `RPCHandler`, or
`/rpc` routes unless the transport strategy changes for the whole app.

## URLs

- Scalar API reference: `/api/docs`
- OpenAPI JSON: `/api/openapi.json`
- API base URL: `/api`

## Adding A Procedure Module

1. Define the contract in `server/orpc/contracts/<module>.ts`.
   Use `oc.route({ method, path, summary, tags })`, `.input(...)`, and
   `.output(...)`. The contract must not import database tables, auth server
   code, or handler implementation details.

2. Register the contract in `server/orpc/contracts/index.ts`.
   This is the object imported by the browser OpenAPI client, so keep it safe
   for client bundles.

3. Implement handlers in `server/orpc/routers/<module>.ts`.
   Use `implement(moduleContract).$context<...>()`, then apply only the
   middleware needed by each procedure:
   - `dbMiddleware` for public data access
   - `authMiddleware` for signed-in users
   - `requireOrgMiddleware` when an active organization is required

4. Register the implemented router in `server/orpc/routers/index.ts`.

5. Consume from React with `orpcQuery.<module>.<procedure>.queryOptions()` or
   `.mutationOptions()` so TanStack Query keys stay generated by oRPC.

## Vike Notes

Keep server-only code behind the Hono/Vike server boundary. Components and
browser clients should import only `server/orpc/client/*` and contract-safe
modules. Route guards should redirect before rendering protected pages, while
procedures must still enforce auth with oRPC middleware.
