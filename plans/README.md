  (`src/zero/mutators.server.ts:101-143`); errors roll back atomically.
- *Per-line tax rounding diverges from totals* (CORRECTNESS-05): totals are
  computed FROM the rounded line values (`create-sale.server.ts:754`), so
  internally consistent; line-level rounding is a legitimate policy choice.

  the check at `register-payment.server.ts:148` is necessary (the join does
  not filter soft-deleted customers) and correct.

  negligible window inside one transaction; fold into any future shifts
  refactor, not worth a plan.
