## Findings considered and rejected

- *Sale cancellation leaves partial state on failure* (audit CORRECTNESS-04):
  wrong — all server mutators run inside Zero's wrapped Drizzle transaction
  (`src/zero/mutators.server.ts:101-143`); errors roll back atomically.
- *Per-line tax rounding diverges from totals* (CORRECTNESS-05): totals are
  computed FROM the rounded line values (`create-sale.server.ts:754`), so
  internally consistent; line-level rounding is a legitimate policy choice.
- *Redundant deleted-customer check in credit payments* (CORRECTNESS-07):
  the check at `register-payment.server.ts:148` is necessary (the join does
  not filter soft-deleted customers) and correct.
- *Shift closure double-read race* (CORRECTNESS-03): real pattern,
  negligible window inside one transaction; fold into any future shifts
  refactor, not worth a plan.
- *.env secret committed* (SEC-01): `.env` is untracked and gitignored;
  hygiene advisory only — rotate any dev secrets that were real.
- *role:null privilege escalation on mutators* (SEC-05): privileged surfaces
  are properly guarded server-side — settings require admin/owner with a
  DB-fetched role (`server/settings/update-settings.server.ts:43-84`),
  member management uses `assertCanManageAccess`
  (`server/organization/organization-mutations.server.ts:120-139`).
  Member-level access to POS/product mutations is by-design.
- *NODE_ENV-conditional secure cookies* (SEC-03), *QZ sign rate-limit*
  (SEC-04), *CORS hardening* (SEC-06): standard patterns / defense-in-depth
  noise; QZ `/sign` is already session-gated by design
  (`server/qz/handler.server.ts:147-151`).
- *posCatalog 1000-row cap too low* (PERF-03): speculative — no evidence of
  >1000-product orgs; search covers the tail.
- *`schemas/` folder restructure* (DEBT-03): structural taste; no observed
  duplicate/conflicting schemas.
- *MIGRATION_PLAN.md stale* (DOCS-01): addressed — doc now reflects completed
  migration and points to AGENTS.md for ongoing conventions.
- *KDS is stubbed* (DIR-02): wrong — a working kitchen board exists
  (`pages/(app)/kitchen/+Page.tsx`, `queries.restaurants.kitchenBoard`).
- *Vike pre-1.0 / Zero vendor lock-in* (DEPS-02/03): watch items, no action.
- *@medibase/webusb-receipt-printer missing* (DX-02): local-environment
  artifact — the package is in `bun.lock`; a fresh `bun install` fixes the
  single `tsc` error.
